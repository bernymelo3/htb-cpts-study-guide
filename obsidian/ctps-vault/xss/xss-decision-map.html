XSS type decision map

CPTS // Cross-Site Scripting module

2-step identification procedure
1

Open DevTools CTRL+Shift+I → Network tab. Inject payload. If no HTTP request fires and URL has # → DOM XSS

2

If request was made, refresh the page. Fires again → Stored. Gone → Reflected

critical Stored XSS
  • Payload saved to DB — fires for every visitor
  • Inject once, own everyone (1-to-many)
  • Visible in page source via CTRL+U
  • Requires DB cleanup to remove
  • Common sinks: comment fields, profiles, to-do lists
medium Reflected XSS
  • Server echoes input in response (error msg, search result)
  • Payload rides in URL via GET param
  • Must trick victim into clicking crafted URL
  • Shows in server logs — WAFs can detect it
  • Copy URL from address bar after injecting = weaponized link
medium DOM XSS
  • 100% client-side — server never sees the payload
  • Input via # fragment (never sent to server)
  • innerHTML blocks <script> → use <img onerror>
  • Invisible to WAFs, server logs, IDS
  • Read script.js to find Source → Sink chain
  • Common sinks: innerHTML, document.write(), outerHTML, jQuery append()
Impact ranking

Stored > Reflected > DOM

Stored = fire-and-forget, mass impact.
Reflected = one click, one victim, server-visible.
DOM = one click, one victim, server-invisible.
