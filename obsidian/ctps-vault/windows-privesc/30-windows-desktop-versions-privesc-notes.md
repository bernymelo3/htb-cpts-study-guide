## What This Section Covers
Attacking end-of-life Windows 7 systems using Windows-Exploit-Suggester to cross-reference installed hotfixes against the Microsoft vulnerability database, then exploiting MS16-032 (Secondary Logon Handle) with a PowerShell PoC to spawn a SYSTEM shell. Windows 7 lacks Credential Guard, Remote Credential Guard, Device Guard, and full AppLocker — making it highly exposed.

## Methodology
1. RDP into the target: `xfreerdp /v:<TARGET_IP> /u:htb-student /p:HTB_@cademy_stdnt!`
2. Open PowerShell and run `systeminfo` — copy the full output to a file on your attacker box (e.g. `systemInfo.txt`)
3. On attacker box, fetch Windows-Exploit-Suggester: `wget https://raw.githubusercontent.com/AonCyberLabs/Windows-Exploit-Suggester/master/windows-exploit-suggester.py`
4. Switch to Python 2.7: `pyenv shell 2.7`
5. Install dependencies: `setuptools-2.0` and `xlrd-1.0.0` (download tarballs, extract, `python2.7 setup.py install`)
6. Update the vulnerability database: `python2.7 windows-exploit-suggester.py --update` — creates a dated `.xls` file
7. Run the suggester: `python2.7 windows-exploit-suggester.py --database <YYYY-MM-DD>-mssb.xls --systeminfo systemInfo.txt`
8. Review output — filter out DoS exploits and irrelevant OS matches; MS16-032 stands out as a reliable local privesc
9. On the target, open PowerShell, navigate to `C:\Tools`, bypass execution policy: `Set-ExecutionPolicy bypass -scope process`
10. Import and run the exploit: `Import-Module .\Invoke-MS16-032.ps1` then `Invoke-MS16-032`
11. A SYSTEM-level CMD window spawns — read the flag: `type C:\Users\Administrator\Desktop\flag.txt`

## Key Takeaways
- Windows-Exploit-Suggester works offline on your attacker box — just feed it `systeminfo` output and it cross-references against the MS vulnerability DB
- It requires Python 2.7 with `setuptools` and `xlrd` — use `pyenv shell 2.7` on Pwnbox
- The tool output is noisy — you need to manually filter out DoS exploits and irrelevant IE/browser vulns to find actionable local privesc paths
- MS16-032 (Secondary Logon Handle) is a reliable privesc on Windows 7 SP1 and Server 2008/2012 — the PowerShell PoC (`Invoke-MS16-032.ps1`) spawns a SYSTEM CMD window directly, no Metasploit needed
- Windows 7 is still estimated to have 100M+ users — common in retail (POS systems), healthcare, education, and government
- The Metasploit `local_exploit_suggester` post module is the in-session alternative when you already have a Meterpreter shell

## Gotchas
- Windows-Exploit-Suggester **only works with Python 2.7** — Python 3 will error out
- The database file name includes the date (`2022-11-22-mssb.xls`) — use the exact filename generated by `--update`
- The `systeminfo` output must be saved with UTF-8 encoding or the parser may choke
- MS16-032 requires the system to have 2+ CPU cores — it's a race condition exploit that won't work on single-core VMs
- The Invoke-MS16-032 PoC spawns a new CMD window as SYSTEM — if you're over RDP this is fine, but over a reverse shell you'd need the Metasploit module version instead
